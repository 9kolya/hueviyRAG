# Internal Document RAG

Сервис индексирует фотографии документов, извлекает из них текст через VLM, режет текст на чанки, сохраняет эмбеддинги в Qdrant и предоставляет HTTP API для инкрементальной дозагрузки и поиска.

## Что уже реализовано

- загрузка изображений через API или чтение из `data/raw`
- OCR/транскрибация изображений через OpenAI-compatible VLM API
- multilingual эмбеддинги через SentenceTransformers
- хранение чанков и метаданных в Qdrant
- инкрементальная индексация: повторно обрабатываются только новые или изменённые файлы
- сохранение извлечённого текста в `data/processed`

## Быстрый старт

1. Установите зависимости:

```bash
pip install -r requirements.txt
```

2. Скопируйте `.env.example` в `.env` и настройте:

- `QDRANT_URL` для вашего контейнера Qdrant
- `VLM_BASE_URL`, `VLM_API_KEY`, `VLM_MODEL` для OCR-модели. По умолчанию проект настроен на OpenAI Vision
- опционально `ANSWER_*`, если нужен генеративный ответ поверх retrieval

3. Запустите API:

```bash
uvicorn src.main:app --reload
```

## Эндпоинты

- `GET /health` проверка сервиса
- `POST /documents/sync` индексация новых и изменённых файлов из `data/raw`
- `POST /documents/upload` загрузка новых изображений и немедленная индексация
- `GET /documents` список документов и статусов индексации
- `POST /query` retrieval и генерация ответа через LLM с указанием источников

## Локальная конфигурация

- `.env` не должен попадать в git
- для старта достаточно задать ключ OpenAI и доступный Qdrant
- если Qdrant уже поднят в Docker, проверьте что `QDRANT_URL` указывает на опубликованный порт контейнера

## Пример запроса

```bash
curl -X POST http://localhost:8000/query \
  -H "Content-Type: application/json" \
  -d '{"question":"Какие условия оплаты указаны в договоре?","limit":5}'
```
